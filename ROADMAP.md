# Kedama Design System — 構築ロードマップ

> **技術スタック:** React + TypeScript  
> **規模:** 小規模（基本トークン + 基礎コンポーネント）  
> **ステータス:** ゼロから構築

---

## 全体の流れ

デザインシステムの構築は、大きく **6つのフェーズ** に分かれます。
各フェーズは前のフェーズの成果物に依存するため、順番に進めていきます。

---

## Phase 1: デザイン原則の策定

デザインシステム全体の「なぜそうするのか」を定める土台です。
コンポーネントを作り始める前に、チームで共有する判断基準を言語化します。

**やること:**

- ブランドの価値観・トーンを整理する（例: 親しみやすい、プロフェッショナル、etc.）
- デザイン原則を3〜5個に言語化する（例: 「シンプルさを優先する」「一貫性を保つ」）
- 対象プロダクト・ユーザーの確認

**成果物:** `docs/design-principles.md`


---

## Phase 2: デザイントークンの定義

色・フォント・余白などの「最小単位の値」を定義します。
トークンはデザインシステム全体の一貫性の源になります。

**やること:**

- **カラーパレット**: プライマリ、セカンダリ、ニュートラル、セマンティック（success, warning, error, info）
- **タイポグラフィ**: フォントファミリー、サイズスケール（xs〜2xl）、ウェイト、行間
- **スペーシング**: 4px基準のスケール（4, 8, 12, 16, 24, 32, 48, 64）
- **角丸（Border Radius）**: none, sm, md, lg, full
- **シャドウ**: sm, md, lg
- **ブレイクポイント**: sm, md, lg, xl

**成果物:** `src/tokens/` ディレクトリ配下にTypeScriptで定義


---

## Phase 3: プロジェクトセットアップ

実際にコードを書き始めるための環境を構築します。

**やること:**

- パッケージマネージャの選定（npm / pnpm / yarn）
- プロジェクト初期化（Vite or Next.js ベース）
- TypeScript設定
- ESLint + Prettier 設定
- Storybookの導入（コンポーネントカタログ用）
- ディレクトリ構造の決定

**想定ディレクトリ構造:**

```
kedama-design-system/
├── src/
│   ├── tokens/          # デザイントークン（色・フォント・余白等）
│   ├── components/      # UIコンポーネント
│   │   ├── Button/
│   │   │   ├── Button.tsx
│   │   │   ├── Button.stories.tsx
│   │   │   ├── Button.test.tsx
│   │   │   └── index.ts
│   │   └── ...
│   ├── styles/          # グローバルスタイル・リセットCSS
│   └── index.ts         # パブリックAPI（エクスポート）
├── docs/                # ドキュメント
├── .storybook/          # Storybook設定
├── package.json
└── tsconfig.json
```

**成果物:** 動作する開発環境 + Storybook


---

## Phase 4: 基礎コンポーネントの実装

トークンを活用して、最低限必要なUIコンポーネントを作ります。
小規模スコープでは、以下の5〜8コンポーネントが目安です。

**優先度順:**

| 優先度 | コンポーネント | 説明 |
|--------|---------------|------|
| 1 | Button | primary / secondary / ghost / danger バリアント |
| 2 | Input | テキスト入力、バリデーション状態対応 |
| 3 | Typography | 見出し・本文・キャプション等のテキスト表示 |
| 4 | Badge | ステータス表示用ラベル |
| 5 | Card | コンテンツをグルーピングするコンテナ |
| 6 | Modal / Dialog | オーバーレイ表示 |
| 7 | Icon | アイコンシステム（SVGベース） |

**各コンポーネントに含めるもの:**

- TypeScriptの型定義（Props）
- アクセシビリティ対応（ARIA属性、キーボード操作）
- Storybookストーリー（バリアント・状態の網羅）
- 基本的なユニットテスト

**成果物:** `src/components/` 配下に各コンポーネント


---

## Phase 5: ドキュメント整備

作ったものを他の開発者やデザイナーが使えるように、ドキュメントを整えます。

**やること:**

- 各コンポーネントの使い方ガイド（Props一覧、使用例）
- デザイントークンの一覧ページ
- インストール・セットアップ手順
- コントリビューションガイド（コンポーネント追加の手順）

**成果物:** Storybookをドキュメントサイトとして活用、または別途ドキュメントサイト


---

## Phase 6: パッケージング・配布

プロダクトから利用できる形にパッケージ化します。

**やること:**

- ビルド設定（ESM / CJS 出力）
- package.json の exports 設定
- npm パッケージとして公開（プライベートレジストリ or npm）
- バージョニング方針の決定（セマンティックバージョニング推奨）
- CHANGELOG の運用方針

**成果物:** `npm install @kedama/design-system` で利用可能な状態


---

## 推奨する進め方

1. **Phase 1〜2 を最初の1〜2週間で完了させる** — ここが土台になるため、じっくり検討する価値があります
2. **Phase 3 は1〜2日で完了できます** — テンプレートやツールの力を借りて素早く
3. **Phase 4 は反復的に進める** — Buttonから始めて、1コンポーネントずつ丁寧に作る
4. **Phase 5〜6 は Phase 4 と並行して少しずつ** — コンポーネントができるたびにドキュメントも書く

---

## 次のステップ

Phase 1「デザイン原則の策定」から始めましょう。
Kedamaのブランドやプロダクトの方向性について話しながら、一緒にデザイン原則を固めていきます。
